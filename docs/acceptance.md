# Software Garden MVP 验收标准

## 启动

- 使用 `npm start` 可以启动本地服务。
- 浏览器访问本地服务端口，例如 `http://localhost:5174`，可以看到首页。
- 不需要安装 MySQL、PostgreSQL 或其他外部数据库。
- SQLite 数据保存在 `data/db.sqlite`。

## 软件创建

- 用户输入一句自然语言需求后，系统创建一个软件。
- 未配置 API Key 时，系统使用 Mock AI 生成软件包。
- 软件包必须包含 `manifest`、`schema`、`ui`、`actions`、`prompts`。
- 生成结果必须经过协议校验后才保存。
- 新增软件过程必须展示过程日志，至少包含需求接收、生成、校验、安装结果。

## Runtime 运行

- 首页布局清爽、信息层级明确，包含顶部导航、能力标签、软件列表、运行区和右侧助手。
- Runtime 三栏布局在桌面端不遮挡、不溢出，移动端自动单列显示。
- Runtime 能根据 `schema.json` 渲染表单和列表。
- Runtime 能根据 `ui.json` 渲染列表、图表、编辑器和仪表盘页面。
- 用户可以新增、编辑、删除、搜索记录。
- 用户可以导出当前实体的 CSV。
- 数据列表字段支持显示/隐藏配置。
- 数据列表支持点击表头按字段升序/降序排序。
- 数据列表支持行内编辑并保存。
- 新增/编辑数据表单支持配置字段顺序。
- 新增/编辑数据表单支持一行 2 列、3 列、4 列布局。
- 表单字段顺序支持拖拽调整，也支持上移/下移按钮。

## AI 修改

- 用户可以在应用内通过右侧助手输入修改需求。
- 修改输出 Patch，而不是整包重写。
- Patch 应用后必须重新校验完整软件包。
- 验收修改命令包括：增加旅游预算功能、增加今日任务页面、增加爆款标题分析、增加跟进提醒日期。
- 修改软件过程必须展示过程日志，至少包含需求接收、Patch 生成、Patch 应用、校验、保存结果。

## Action

- 支持内置 Action runner。
- 支持 `ai.generateText`、`data.queryRecords`、`export.markdown`、`export.csv`。
- MVP 不执行 AI 生成的任意代码。

## 导入导出

- 应用可以导出为 `.sgpkg`。
- `.sgpkg` 是 zip 包，包含 `manifest.json`、`schema.json`、`ui.json`、`actions.json`、`prompts.json`。
- 导入 `.sgpkg` 后能安装为一个新的应用。
- 默认导出应用结构，不导出真实用户数据。

## 四个验收用例

- 家庭记账本：列表、表单、分类统计、CSV 导出、旅游预算 Patch。
- ToDoList：任务列表、新增任务、完成状态、搜索、今日任务 Patch。
- 公众号文章生成器：文章列表、AI 生成文章 Action、Markdown 导出、爆款标题分析 Patch。
- 客户管理器：客户列表、新增编辑删除、状态统计、CSV 导出、跟进提醒日期 Patch。
